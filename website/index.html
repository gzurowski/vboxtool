VBoxTool

Easy control of virtual machines of VirtualBox  (virtualization solution) on a Linux headless server, published as free and open source software.

Currently VirtualBox lacks a decent management environment for controlling virtual sessions on a headless server environment. VBoxTool mimics partly Virtual Machine Manager which controls sessions for other virtualization solutions like qemu, kvm, etc. Unfortunately, VirtualBox is not in the list of supported engines (nor will be in the near future).

Heart of the framework is a script with which it is possible to start, stop, save, backup and show status of registered sessions in batch mode; execution is by command line. Next to develop is a webserver which points to the control script. With this you can activate all functions from the script, all by a web page.

Usage

VBoxTool currently consist only of a script. With this script, virtual machines of VirtualBox can be controlled: start, stop, save, backup and show status of sessions in batch mode from the command line. Several options like session to start, vrdp port, etc. can be controlled by a configuration file.

~$ vbox --help
Usage: vbox OPTION [session]
Show info about VirtualBox sessions or control those sessions.

Options:
  show              Show status of all sessions.
  showrun           Only show status of running sessions.
  start [session]   Start all saved sessions or only the given session.
                    When no session name is given, all saved sessions will be
                    started; powered off and aborted sessions are left alone.
  autostart         Starts all sessions in a predefined configuration file.
  save [session]    Save all running sessions or only the given session.
  stop [session]    Stop all running sessions or only the given session.
  backup [session]  Backup all running sessions or only the given session.
                    Running sessions are saved and started after backup has
                    completed.
  --version|version Version info.
  --help|help       This help.

Through autostart, sessions can be started in a controlled way: only the
sessions in the configuration file will be started. As a bonus, the vrdp port
(also configurable) can be set at startup time.
Option autostart depends on a configuration file, /etc/vbox/machines.conf.
Each line in this file is a separate machine. Structure of each line:
<session-name>,<vrdp-port>. The given vrdp-port is set statically to the
session, prior to starting; state is discarded when session is in savestate.

Example for /etc/vbox/machines.conf:
Ubuntu Desktop,3391
Ubuntu JeOS,3392
Do not use spaces before and after the ',' delimiter.

Saving sessions is preferred above stopping: this is faster when restoring
and safer because session can appear to be cold booted.

Facts

  • Minimal requirements:
    • Linux (kernel & distro agnostic, although developed and tested only on Ubuntu)
    • VirtualBox 1.6.x
  • Project status: Beta. Script is available for testing purposes.
